# REQ_004 — 리얼타임 채팅 경험 (즉시 화면 업데이트)

## 요구사항

HTTP(7777) 요청이나 프롬프트 입력이 들어왔을 때, 서버 응답을 기다리지 않고 **즉시** 화면을 전환하여 유저 메시지를 보여준다. AI 응답은 나중에 도착하는 대로 화면에 추가한다.

### 현재 상태 (동기적)
1. `_handleSend` 호출
2. `await _chatService.sendMessage(text)` (응답 올 때까지 멈춤)
3. 응답 도착
4. 유저 메시지 + AI 응답을 리스트에 넣고 `_pushScreen` 호출

### 개선된 상태 (비동기적/단계적)
1. `_handleSend` 호출
2. **[즉시]** 유저 메시지를 `_messages`에 추가
3. **[즉시]** `_pushScreen` 호출 → 채팅창에 내 질문이 먼저 뜸
4. **[대기]** `await _chatService.sendMessage(text)`
5. 응답 도착
6. **[업데이트]** AI 응답을 `_messages`에 추가
7. **[업데이트]** 다시 `_pushScreen` 호출 → 채팅창에 답변이 업데이트됨

## AI 구현 프롬프트

### PROMPT START

You are a Flutter developer. Modify `lib/screens/tizen_chat_screen_2.dart` to provide immediate visual feedback.

**Goal**: When a message is sent (via PromptBar or HTTP), the UI should transition to the chat screen **immediately** showing the user's message, while the AI response is fetched in the background.

---

### Modification: `_handleSend(String text)` in `_TizenChatScreen2State`

Rewrite the `_handleSend` method to perform a two-stage update.

**Stage 1: Pre-response update**
1. Request focus for keyboard.
2. Set UI states (`_isVisible = false`, etc.).
3. Add the User message to `_messages` **immediately**.
4. Call `_pushScreen` with the current messages (User only). This ensures the user sees their message right away.
5. Set `_isWaiting = true`.

**Stage 2: Post-response update**
1. Await the AI response in the background.
2. Once the response arrives, extract the `rawText` and `rawUiCode`.
3. Add the AI message to `_messages`.
4. Set `_isWaiting = false`.
5. Call `_pushScreen` **again** with the updated messages. Because of our `REQ_003` logic (pop and push), this will seamlessly update the chat view with the AI's answer.

#### Implementation Detail for `_handleSend`:

```dart
Future<void> _handleSend(String text) async {
  _keyboardFocusNode.requestFocus();

  setState(() {
    _shouldSlideDown = false;
    _isVisible = false;
    _responseMessage = "";
  });

  // Small delay for clean animation transition
  await Future.delayed(const Duration(milliseconds: 200));
  if (mounted) setState(() => _shouldSlideDown = true);
  await Future.delayed(const Duration(milliseconds: 100));

  if (!mounted) return;

  // STEP 1: Add user message and push screen IMMEDIATELY
  setState(() {
    _messages.add(ChatMessage(text: text, type: MessageType.sent));
    _isWaiting = true;
  });

  // Open the screen showing only the user message first
  _pushScreen(TizenChatScreen(initialMessages: List.from(_messages)));

  try {
    // STEP 2: Fetch AI response in the background
    final response = await _chatService.sendMessage(text);

    if (mounted) {
      // Extract logic (rawText, rawUiCode fallback as before)
      final String rawText;
      if (response['text'] != null && response['text'].toString().isNotEmpty) {
        rawText = response['text'].toString();
      } // ... use all existing fallback logic ...
      else { rawText = '응답 없음'; }

      final dynamic rawUiCode = response['ui_code'];

      setState(() {
        _isWaiting = false;
        _currentText = rawText;

        final chatMsg = ChatMessage(
          text: _currentText,
          type: MessageType.received,
          uiCode: (rawUiCode != null && rawUiCode.toString().isNotEmpty)
              ? rawUiCode.toString()
              : null,
        );
        _messages.add(chatMsg);

        // STEP 3: Push updated content (this will pop the user-only screen and push the full one)
        if (chatMsg.uiCode != null) {
          _pushScreen(GenerativeUIScreen(text: _currentText, uiCode: chatMsg.uiCode!));
        } else {
          _pushScreen(TizenChatScreen(initialMessages: List.from(_messages)));
        }
      });
    }
  } catch (e) {
    if (mounted) {
      setState(() {
        _isWaiting = false;
        _responseMessage = "오류 발생: ${e.toString()}";
      });
    }
  }
}
```
