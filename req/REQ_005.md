# REQ_005 — TizenChatScreen2 전면 재구성

## 현재 코드 문제점 분석

### (A) 키 매핑 불일치
- `ArrowUp` → `_pushHttpMessageOverlay()` (잘못됨, 요구사항은 `ArrowDown`)
- `Alt/Meta` → `_toggleVisibility()` (요구사항은 `ArrowUp`)

### (B) HTTP 라우팅 로직 혼재
- 오버레이(`_isOverlayActive`) 중에는 HTTP 메시지를 차단하는데, 오버레이가 직접 `HttpMessageBus`를 구독하므로 이중 처리 불필요
- `_isChatScreenActive` 플래그와 `pop → .then()` 간의 레이스 컨디션이 여전히 잠재

### (C) UI 불명확
- 첫 화면이 완전히 투명 → 앱이 떠 있는지 사용자가 알 수 없음
- 요구사항: 오른쪽 상단 녹색 동그라미로 앱 활성 상태 표시

### (D) 불필요한 복잡도
- `ScreenState`, `_activeScreen`, `_currentText` 필드가 실제로 화면 전환에 사용되지 않음
- `_isChatScreenActive` 플래그의 레이스 컨디션 구조_가 복잡함

---

## 새 요구사항 정의

| 입력 | 동작 |
|------|------|
| `ArrowDown` 키 | `HttpMessageOverlayScreen` push |
| `ArrowUp` 키 | 프롬프트창 토글 (`_toggleVisibility`) |
| HTTP 7777 (`_isOverlayActive = true`) | 오버레이가 직접 구독하므로 **라우팅 불필요** |
| HTTP 7777 (`_isVisible = true`, 프롬프트 열림) | `_handleSend(text)` 호출 → AI 처리 |
| HTTP 7777 (그 외 = 기본 상태) | 무시 |

### 첫 화면 UI
- 화면 오른쪽 상단에 **직경 16px 녹색 원(#4CAF50)** 표시
- 나머지는 완전 투명 유지

### `_handleSend` 동작 (프롬프트 전용)
1. 프롬프트 숨기기 애니메이션
2. **즉시** 유저 메시지 → `_messages.add(...)` → `_pushScreen(TizenChatScreen)`
3. 백그라운드에서 AI 응답 수신
4. 응답 도착 → `_messages.add(aiMsg)` → 화면 갱신

#### `_handleSend` 2단계 비동기
```dart
Future<void> _handleSend(String text) async {
  _keyboardFocusNode.requestFocus();

  // 프롬프트 숨기기
  setState(() { _shouldSlideDown = false; _isVisible = false; _responseMessage = ""; });
  await Future.delayed(const Duration(milliseconds: 200));
  if (!mounted) return;
  setState(() { _shouldSlideDown = true; });
  await Future.delayed(const Duration(milliseconds: 100));
  if (!mounted) return;

  // STEP 1: 유저 메시지 즉시 추가 + 채팅창 열기
  setState(() {
    _messages.add(ChatMessage(text: text, type: MessageType.sent));
    _isWaiting = true;
  });
  _pushChatScreen();

  // STEP 2: AI 응답 백그라운드 처리
  try {
    final response = await _chatService.sendMessage(text);
    if (!mounted) return;

    final rawText = _extractText(response);
    final rawUiCode = response['ui_code']?.toString();

    setState(() {
      _isWaiting = false;
      _messages.add(ChatMessage(
        text: rawText,
        type: MessageType.received,
        uiCode: (rawUiCode?.isNotEmpty == true) ? rawUiCode : null,
      ));
    });

    // STEP 3: 화면 갱신
    final lastMsg = _messages.last;
    if (lastMsg.uiCode != null) {
      _refreshScreen(GenerativeUIScreen(text: rawText, uiCode: lastMsg.uiCode!));
    } else {
      _refreshChatScreen();
    }
  } catch (e) {
    if (mounted) {
      setState(() { _isWaiting = false; _responseMessage = "오류: ${e.toString()}"; });
      Future.delayed(const Duration(seconds: 3), () {
        if (mounted) setState(() { _responseMessage = ""; });
      });
    }
  }
}

String _extractText(Map<String, dynamic> response) {
  for (final key in ['text', 'response', 'content', 'message', 'response_text']) {
    final val = response[key]?.toString();
    if (val != null && val.isNotEmpty) return val;
  }
  return '에이전트로부터 응답을 받지 못했습니다. (Empty response)';
}
```

      _isPromptScreenActive = false;
      Navigator.of(context).pop();

    _isPromptScreenActive = true;
    Navigator.of(context).push(

  Future<void> _handleSend(String text) async {
    _keyboardFocusNode.requestFocus();

    // Hide prompt bar with animation
    setState(() {
      _shouldSlideDown = false;
      _isVisible = false;
      _responseMessage = "";
