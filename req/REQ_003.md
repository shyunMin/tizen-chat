# REQ_003 — HTTP 입력 대화 누적 (프롬프트 동일 플로우)

## 문제 정의

HTTP(7777)로 들어온 메시지가 대화로 쌓이지 않고 매번 새로 시작된다.

### 근본 원인 분석 (`tizen_chat_screen_2.dart`)

#### 원인 1 — 히스토리 초기화 (L110)
```dart
// _handleSend() 내부
_messages.clear();  // ← 매 호출마다 이전 대화 삭제
_messages.add(ChatMessage(text: text, type: MessageType.sent));
```
매번 새로운 1:1 쌍만 가지고 `TizenChatScreen`을 push → 대화가 누적되지 않음.

#### 원인 2 — HTTP 수신 조건 `_isVisible` (L49)
```dart
if (mounted && _isVisible && !_isWaiting && !_isOverlayActive) {
  _handleSend(text);
}
```
첫 번째 HTTP 메시지를 처리하면 `_isVisible = false`로 바뀌므로
이후 들어오는 모든 HTTP 메시지가 무시됨 → 첫 메시지 이후 대화 불가.

---

## 원하는 동작

1. HTTP로 메시지가 반복적으로 들어오면 대화가 누적되어야 한다.
2. 이미 대화창이 열려 있으면 닫고, 누적된 히스토리를 포함한 새 대화창을 push한다.
3. 키보드(프롬프트)와 완전히 동일한 흐름으로 처리된다.

---

## 설계

### 상태 추가
```
bool _isChatScreenActive = false
  - _pushScreen() 진입 시 true
  - .then() 콜백 시 false
```

### HTTP 리스너 조건 변경
```
// Before
if (mounted && _isVisible && !_isWaiting && !_isOverlayActive)

// After
if (mounted && !_isWaiting && !_isOverlayActive)
```
프롬프트가 보이든 안 보이든 HTTP 메시지를 받는다.

### _handleSend() 수정
```
// Before (L110)
_messages.clear();

// After — 누적 유지, 단 chat 화면에서 직접 돌아온 경우는 .then()이 클리어
// 해당 줄 삭제
```

### _pushScreen() 수정
기존 채팅 화면이 활성화된 상태(`_isChatScreenActive = true`)에서 새 메시지가 오면
현재 화면을 먼저 `pop` 후 최신 히스토리로 다시 `push`한다.

```dart
void _pushScreen(Widget screen) {
  if (_isChatScreenActive) {
    // 기존 대화 창 닫고 재진입 (히스토리 유지)
    Navigator.of(context).pop();
  }
  setState(() {
    _isChatScreenActive = true;
    _isVisible = true;
    _shouldSlideDown = true;
  });
  Navigator.of(context).push(...).then((_) {
    if (mounted) {
      setState(() {
        _isChatScreenActive = false;
        _messages.clear(); // 완전히 닫을 때만 클리어
      });
    }
  });
}
```
